# Proposal A Draft

## Toward a Causal Microstructure of Prediction Market Calibration

Draft status: first prose draft grounded in the current `polymarket` repo. This
is a submission scaffold, not a finished camera-ready manuscript. Result slots
are marked explicitly where the experiments still need to be run.

## Abstract

Prediction markets are often evaluated descriptively: a market is called
"well-calibrated" when prices match realized frequencies on average. That view
is useful but incomplete. Calibration is not a primitive property of a market;
it is an outcome generated by liquidity, participation, timing, category
structure, and the way information arrives. We propose a causal study of those
drivers using a continuously collected Polymarket research stack. The current
repository already supports market ingestion, trade aggregation, calibration
tracking, market microstructure computation, and several causal-analysis
modules. A checked-in audit of the resolved-market dataset reports 11,368
resolved markets, 5,290 resolved markets with trade histories, and a smaller
high-frequency subset with orderbook coverage. Building on that substrate, we
define calibration targets from pre-resolution prices, engineer trade- and
liquidity-based market features, and estimate which features have direct effects
on forecasting quality rather than merely correlating with it. The paper's core
claim is that prediction-market calibration should be modeled as a causal
outcome with domain-specific mechanisms, not as a single pooled descriptive
statistic. The strongest current path is a primary analysis on metadata and
trade-derived features, with richer orderbook and wallet analyses on smaller
subsets. This draft specifies the estimands, data slices, figures, and
validation checks needed to turn the existing repo into a workshop-ready paper.

## 1. Introduction

Prediction markets are increasingly treated as live probabilistic forecasts for
politics, macroeconomics, sports, and crypto. Classical work shows that these
markets can be reasonably calibrated, especially near resolution, but it also
shows that calibration varies with horizon and market design. More recent work
argues that calibration is structured and domain-specific rather than uniform
across questions or exchanges. What remains underdeveloped is a causal account
of *why* some markets calibrate well while others fail.

That gap matters for both market science and AI forecasting. If a forecaster
uses market prices as a baseline, it needs to know when the market should be
trusted, when it should be discounted, and which observed market conditions are
mechanisms rather than symptoms. A purely correlational treatment of spread,
volume, or concentration is not enough.

This paper targets that gap. Our central question is:

What market-level and microstructural features causally drive prediction-market
calibration, and how does that structure vary across domains?

### Contributions

1. We reframe prediction-market calibration as an outcome generated by market
   structure and microstructure, rather than as a single descriptive summary.
2. We define a repo-grounded extraction and feature pipeline for resolved
   Polymarket contracts using metadata, trades, and subset microstructure data.
3. We specify a causal workflow that combines calibration analysis,
   domain-stratified structure learning, and treatment-effect estimation.
4. We define a set of diagnostic outputs that can later be consumed by AI
   forecasting agents as trust or discount signals.

## 3. Outcome Definition: Calibration As A Market-Level Target

We want an outcome that can be used in descriptive analysis, causal structure
learning, and later treatment-effect estimation.

The primary contract-level target is the Brier error at a fixed horizon before
resolution:

`Brier_h = (p_h - y)^2`

where `p_h` is the pre-resolution market-implied probability at horizon `h` and
`y` is the realized binary outcome.

The paper should report three closely related targets:

- Contract-level Brier score at multiple horizons.
- Expected Calibration Error (ECE) computed over decile bins.
- Reliability curves stratified by category and time-to-resolution.

Recommended horizon grid for the first run:

- 7 days before resolution
- 24 hours before resolution
- 6 hours before resolution
- final observed pre-resolution price

If some horizons are unavailable because of shallow collection windows, the
paper can collapse to "earliest observed reliable price" plus "final
pre-resolution price" and make that limitation explicit.

## 5. Identification Strategy

The report memo proposed a four-stage workflow. The repo already implements part
of it, but the first submission should narrow claims to what the code and data
actually support.

### Stage 1: Descriptive Calibration Decomposition

Before causal estimation, quantify how calibration changes with:

- category
- horizon
- trade intensity
- market duration
- negative-risk structure

This stage yields the first hard results in the paper and motivates the causal
question with concrete heterogeneity rather than broad claims.

### Stage 2: Structure Learning

The repo already contains a simplified PC-style analyzer and complementary
information-flow modules. The first submission can therefore report:

- a graph over observed market-level features and calibration outcomes
- domain-specific graphs for high-volume categories
- sensitivity checks using alternative edge criteria

Important drafting note:

- The memo proposed PC + FCI + GES.
- The current repo clearly contains PC-style discovery, Granger analysis, and
  transfer-entropy analysis.
- GES / FCI can remain planned extensions unless they are actually run before
  submission.

The paper should never imply those methods are implemented if the final code
path still only uses the simplified PC plus complementary directional tests.

### Stage 3: Effect Estimation

After selecting a small number of interpretable treatments with adequate
overlap, estimate their effect on calibration. The most realistic first-pass
treatments are:

- high vs low trade intensity
- high vs low concentration
- short vs long time-to-resolution regime
- high vs low recent information-arrival regime

The natural estimators are doubly robust methods such as AIPW or a DR-learner.
If time is short, the paper can report a lighter-weight design:

- stratified regression with strong controls
- doubly robust estimates for one or two treatments only
- domain-specific re-estimation in politics vs sports vs crypto

### Stage 4: Validation

The validation story should be practical rather than maximalist:

- temporal holdout by resolution date
- category holdout where feasible
- placebo outcomes on structurally unrelated categories
- subset robustness checks without orderbook features

The memo's quasi-experimental design around liquidity incentives remains a good
appendix target, but it should only be included if we can document market-level
assignment cleanly from the data.

## 6. Preliminary Feasibility Story

Even before the final causal estimates are run, the repo already supports a
credible feasibility narrative:

- The system has resolved-market extraction logic.
- The repo already includes calibration tracking code.
- The microstructure engine already computes spread, depth, toxicity, and
  impact measures for active markets.
- Existing embedding work in the repo demonstrates that resolved-market
  lifecycles can be summarized and modeled at scale.

That means Proposal A is not a speculative idea. It is a near-term empirical
paper whose remaining work is concentrated in extraction, calibration slicing,
and effect estimation.

## 7. Honest Limitations

This section should stay in the paper, not just in our working notes.

1. The checked-in audit flags the `market_prices` table as broken in its then
   current state. If that has not been fixed by submission time, price features
   should be derived from trades or OHLCV views instead.
2. Orderbook coverage for resolved markets is sparse in the audited snapshot, so
   any orderbook-heavy conclusions should be subset-specific.
3. Wallet-based participation features are missing for a large fraction of
   resolved markets.
4. Category imbalance is severe. Sports and esports dominate the resolved set,
   so pooled estimates can be misleading.
5. Any causal claims depend on overlap and measurement quality. A short workshop
   paper should present them as "estimated effects under observed controls," not
   as definitive platform-wide laws.
